PROJECT ANALYSIS: MAMBAOCR

1. PROJECT OVERVIEW
This project implements an OCR system using ResNet34 (CNN), Mamba (RNN-alternative), and CTC.
It uses LoRA for efficient fine-tuning.

2. FILES
- train.py: Training script.
- infer.py: Inference script.
- eval.py: Evaluation metrics (CER/WER).
- utils.py: Helpers for metrics and decoding.
- clean_data.py: Cleanup utility.
- gen_data.py: Generates synthetic training data.
- requirements.txt: Dependencies.
- README.md: Documentation.

Configs:
- configs/config.py: Hyperparameters and paths.

Models:
- models/ocr_model.py: Main model.
- models/cnn_backbone.py: ResNet34 with adapters.
- models/mamba_encoder.py: Mamba sequence encoder.

3. CONFIGURATION (configs/config.py)
Defaults:
- Input: 32x320 images.
- Model: ResNet34 (512ch) + Mamba (6 layers).
- Training: Batch 256, FP16 enabled.

4. DATA PIPELINE
- Generation: random fonts, rotation, noise.
- Cleanup: removes generated data.

5. ARCHITECTURE
- Input: [Batch, 3, 32, 320]
- CNN: ResNet34 with adapters -> [Batch, 512, 1, Width]
- Mamba: Models sequence context.
- Classifier: Maps to characters.

6. METRICS
- CER: Character Error Rate.
- WER: Word Error Rate.

7. USAGE
1. Install requirements.
2. Generate data (optional).
3. Train (python train.py).
4. Infer (python infer.py).
